 Finances Tracker (TypeScript Migration)
Этот проект представляет собой приложение для отслеживания финансов, демонстрирующее миграцию с JavaScript (исходник -  https://github.com/MarochkinV/finances_JS.git)  на TypeScript.

Приложение позволяет пользователям:
1.	Управлять категориями доходов и расходов: создавать, редактировать и удалять категории.
2.	Записывать операции: добавлять доходы и расходы с указанием категории, суммы, даты и комментария.
3.	Просматривать список операций с фильтрацией по периоду (сегодня, неделя, месяц, год, интервал).
4.	Визуализировать данные: просматривать диаграммы доходов и расходов.
5.	Аутентифицироваться: регистрироваться, входить и выходить из системы.
Исходный код был полностью переписан с JavaScript на TypeScript

Технологии
1.	TypeScript: Язык программирования, обеспечивающий строгую типизацию поверх JavaScript.
2.	JavaScript (ES6+): для функций, которые ещё не перенесены в TypeScript или где это оправдано.
3.	HTML / SCSS: Структура и стили страниц.
4.	Webpack: Система сборки проекта.
5.	Chart.js: для визуализации финансовых данных.
6.	AdminLTE: HTML-шаблон для админ-панели (стили и структура).
7.	Font Awesome: Библиотека иконок.

Структура проекта
1.	src/: Исходный код приложения.
2.	components/: Компоненты страниц (Login, Main, IncomeCategory и т.д.).
3.	types/: Файлы с определениями типов (interfaces, types).
4.	utils/: Вспомогательные утилиты (HttpUtils, AuthUtils).
5.	styles/: Файлы стилей (SCSS).
6.	app.ts: Точка входа в приложение.
7.	router.ts: Логика роутинга.
8.	dist/: Результат сборки (скомпилированный JS, CSS, HTML, шрифты, изображения). *
9.	node_modules/: Зависимости проекта. *
*Не включен в репозиторий. Устанавливается локально npm install  и npm run dev

Установка и запуск
1.	Клонируйте репозиторий: git clone https://github.com/MarochkinV/finances_TS.git
2.	Перейдите в директорию проекта: cd finances_TS/frontend 
3.	Установите зависимости: npm install
4.	Запустите в режиме разработки: npm run dev 
5.	Соберите проект: npm run build 

*(Примечание: для полной работоспособности требуется бэкенд-сервер, обрабатывающий API-запросы.)
